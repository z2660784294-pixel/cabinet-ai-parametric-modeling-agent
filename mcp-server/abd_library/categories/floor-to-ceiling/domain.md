# 图像分析参考：一柜到顶衣柜

> 该文档提供给 LLM 在图像分析阶段使用。
> 你需要按照下方的分析步骤对图像进行分析，同时该文档定义了单元柜的类型。

---

## 前置规则（必读）

⚠️ 分析柜体**主要结构**时**只关注柜体构件**：

- ✅ 关注：门板、抽屉、开放格、收口板、踢脚、见光板、把手
- ❌ 忽略：装饰品（摆件、相框、花瓶）、衣物（挂起或叠放）、衣架、鞋帽、箱包、床品等收纳物

这些"忽略项"**不影响**立面分格数、单元柜数、门板数的判定。

---

## 前置硬约束（一字到顶类目专属，最高优先级）

⚠️ **默认 1 立面分格 = 1 单元柜**：本类目下每个立面分格**默认映射到 1 个单元柜**。仅当满足下表任一**显式垂直拆分证据**时，才允许拆为 顶柜+底柜 成对结构。

| 显式垂直拆分证据 | 判定要点 | key_features 标记 |
|---|---|---|
| **物理独立柜体边界** | 分格内中间观察到完整的独立柜体外缘——**不是开放层板**或装饰横线，而是上柜底面 + 下柜顶面构成的物理隔断（含柜板厚度 ≥18mm）；上下两段各自闭合独立 | `internal_independent_cabinet_boundary` |
| **用户明确指定** | 用户文本明确要求"上半带门顶柜 + 下半带门底柜" | `applied_rule = user_explicit_request` |

**以下现象不构成拆分证据**——默认作为该单元柜的**内部构造**写入属性字段，**不拆 2 单元柜**：

- 上下材质 / 颜色不同 → 内部装饰变化（不专门记录）
- **上段开放层架 + 下段封闭门柜** → 边柜（最左/最右 + 窄宽）或 高柜/下柜（含开放区）（中部），写 `has_open_shelf=true` + `open_shelf_position=top`
- 上段挂衣 + 下段抽屉 → 高柜/下柜 内部分区，写 `drawer_count`
- 上段开放 + 下段开放 → 开放柜（door_count=0）
- **上下分段门板 / 清晰横向门缝** → 仍是同 1 个单元柜，不拆为顶柜+底柜；但门数按可见独立门板累计到 `door_count`（例如同一 unit 内左侧 1 扇通高门 + 右侧上/下 2 扇门，则 `door_count=3`）

### R-door-1：上下分段门板的门数累计

“上下分段门板 + 装饰横线 / 腰线”不构成顶柜+底柜的垂直拆分证据，仍保持 1 个 unit；但可见独立门板必须计入 `units[].door_count`。

对 `高柜/下柜`：
- 若单个 bay 内上段、下段均为可见封闭门板，则该 bay 计 2 门；
- 若 2 个相邻 bay 合并为 1 个 unit，且每个 bay 均为上下分段门板，则 `door_count = 4`；
- 若 1 个 bay 作为单门收尾 unit，且该 bay 为上下分段门板，则 `door_count = 2`；
- 不得将“上下分段不拆 unit”误写为“door_count 只按横向 bay 数计数”。

**优先级**：本前置硬约束 > §2.3 步骤 A > §主要类型语义 §顶柜/底柜定义；冲突时本约束生效。

## 分析主流程

基于图像分析组合衣柜结构的主要步骤：

> 立面分格 (Elevation Bay) → 柜体单元（单元柜）

### 步骤 1：确定立面分格

**立面分格**：指衣柜正立面上，在水平方向上由主要竖向分割线划分出的近似等宽的竖向区间，这些分格后续进一步组合成单门柜、双门柜、边柜或抽屉柜等单元柜。

#### 立面分格的生成 / 识别方法

1. **提取主竖向分割线**
   主竖向分割线包括：门板之间的竖向缝、开放柜与封闭柜之间的竖向边界、不同材质或不同深度之间的竖向边界、上下门板虽然被横缝打断但仍然对齐的竖向线。这些线决定了立面分格。

2. **剔除非分格线**
   不参与立面分格计数的元素包括：层板横线、抽屉横缝、把手、门板表面装饰线、阴影线、极窄侧封板、收口条。

3. **用"近似等宽"校验分格数量**
   一字到顶衣柜通常有稳定的竖向节奏。判断时应优先寻找一个基础宽度：
   - 一个接近基础宽度的竖向区间 = 1 个立面分格
   - 一个接近两倍基础宽度的区域 = 2 个立面分格
   - 一个接近三倍基础宽度的区域 = 3 个立面分格

   所以，某个区域看起来很宽，但如果它约等于两个常规门宽，就应拆成两个立面分格，而不是当成一个超宽分格。

4. **横向跨格构件标定（必做，强证据来源）**

   在确定立面分格之后、进入步骤 2 合并之前，**必须**对画面中可见的横向构件做一次单独扫描：

   - 候选对象：抽屉、连续把手 / 拉手凹槽带、跨多格的开放层架带、跨多格的连续门板（罕见）
   - 对每个候选构件标定它的**水平左右边界**，并赋一个唯一 `band_id`（如 `DB1`=drawer band 1、`HB1`=handle band 1、`OB1`=open-shelf band 1）
   - **判定跨格**：构件宽度 ≈ k × 基础分格宽（k ≥ 2）且其左右边界横跨 ≥2 个相邻 bay → 标记为"跨格构件"
   - 凡跨 ≥2 bay 的 band，必须在 5a 表对应行的 `key_features` 写入形如 `drawer_band:DB1` / `handle_band:HB1` / `open_shelf_band:OB1` 的标签，并在表中新增 `bands` 列把同一 `band_id` 同时标记到所有它跨过的 bay 上
   - 该标定结果为 §2.1 第 0 条 R-bay-7 提供输入；漏标会直接导致 5b 合并错误

   **不构成跨格构件的反例**（不得标 band_id）：
   - 单格内部的水平装饰横线 / 腰线 / 木纹横条 → 装饰，不是构件
   - 每个分格独立的 J-pull 拉手凹槽 → 各自独立，不是跨格 handle band（仅当凹槽是一根连贯横线穿过 ≥2 bay 才算）
   - 各分格各自独立的抽屉（同一水平高度但**互相之间有竖向分隔线**）→ 各为独立抽屉，不是跨格 drawer band

### 步骤 2：基于立面分格组合成柜体单元（单元柜）

把手、开放柜、抽屉、材质变化主要用于判断"分格如何组合成柜体单元"。

#### 2.1 合并优先级（强约束）

0. **【最高优先级 / 几何硬约束 / R-bay-7】跨格构件 = 同一柜体**：当一个抽屉、一个连续把手 / 拉手凹槽带、一个连续开放层架带、一个连续门板，在水平方向**跨越 ≥2 个相邻立面分格**时（由 §步骤 1.4 标定），这些被跨越的分格**必须**合并为同一个单元柜，`bay_count` = 跨越的分格数；这是单元柜内部物理结构（侧板必须支撑跨格抽屉滑轨 / 跨格门板）的硬约束，**优先级高于本节其余条目、§2.2 单门反证、§2.5 边柜邻接位置偏好、§2.3 贪心方向**。仅当用户文本明确要求拆分（`applied_rule = user_explicit_request`）时才允许覆盖。
   - 典型触发场景：3 个抽屉横跨 2 个分格的下区 → 这两个分格强制合并为 1 个双门柜（即使其中一个分格紧邻边柜也不退化为单门）
1. **两个相邻分格优先组成双门柜**——这是最常见的组合方式，**默认对所有相邻等宽分格触发**；只有出现下方 §2.2 的"显式单门反证"时才退化为单门柜。
2. 一对对称把手通常表示两个相邻分格组成一个双门柜。
3. 开放边柜通常占 1 个立面分格。
4. 带抽屉的柜体可能占 1 个或 2 个立面分格，要看它的宽度节奏；**跨格抽屉**直接走第 0 条 R-bay-7。
5. 单侧把手通常表示一个单门柜。
6. 上下都开放的开放柜单独生成柜体单元，一般占 1 个立面分格。

#### 2.2 显式单门反证（只有以下任一存在才能拆为单门）

| 反证 | 说明 | 对应 `merge_skip_reason` |
|---|---|---|
| 把手对称性破缺 | 每个分格各自独立把手、不成对 | `handle_asymmetry_break` |
| 材质 / 深度突变 | 相邻分格颜色 / 木纹 / 深度明显不同 | `material_or_depth_break` |
| 相邻分格为开放柜 | 一格为开放柜则另一格独立 | `adjacent_open_shelf` |
| 相邻分格为抽屉柜 | 抽屉柜本身独立、邻格不与之合并 | `adjacent_drawer_unit` |
| 奇数分格剩余 | N 为奇数，已优先合并双门后剩 1 格 | `odd_bay_remainder` |
| 单分格组合 | 组合柜总分格数 N=1 | `single_bay_total` |
| 用户明确指令 | 用户文本要求"逐分格独立成柜" | `user_explicit_request` |

⚠️ **当上述反证均不存在时（典型场景：3 扇等宽白门、按压式无可见把手、无开放格 / 抽屉 / 材质突变），仍必须走"优先双门"分支**——不允许"看不出线索就全拆单门"。

#### 2.3 合并决策树（伪代码 / 强制执行）

```text
function decompose_bays_to_units(N_bays, key_features, edge_cabinet_at, cross_bay_bands):
    """
    edge_cabinet_at: 'left' | 'right' | 'both' | 'none'
        门柜序列两端是否邻接边柜（type=边柜）。由前序边柜识别提供。
    cross_bay_bands: list of {band_id, kind ∈ {drawer|handle|open_shelf|panel}, bays:[i,j,...]}
        §步骤 1.4 标定的所有跨 ≥2 bay 的横向构件。
    """
    if N_bays == 0:
        return []
    if N_bays == 1:
        return [ Unit(bay_count=1, skip_reason='single_bay_total') ]

    # 步骤 A0（最高优先级 / R-bay-7 跨格构件锚定 / §2.1 第 0 条）：
    # 先消费所有跨格构件——把它跨过的相邻 bays 直接锁定为同一 unit（bay_count = len(bays)）。
    # 这些 unit 不再参与后续 §2.2 反证 / §2.5 方向偏好判定。
    # 仅 'user_explicit_request' 可覆盖该锁定。
    locked_units = []
    for band in cross_bay_bands:
        if user_explicit_request_to_split(band):
            continue
        locked_units.append( Unit(bays=band.bays, bay_count=len(band.bays),
                                  applied_rule='preferred_double_door',
                                  evidence='cross_bay_band:' + band.band_id) )
    remaining_bays = bays_not_locked(N_bays, locked_units)

    # 步骤 A（垂直拆分判定 / 默认不拆）：
    # 一字到顶类目下，仅当出现以下任一显式拆分证据时，才把分格拆为 顶柜+底柜 成对：
    #   - key_features 含 'internal_independent_cabinet_boundary'（物理独立柜体边界，含柜板厚度 ≥18mm）
    #   - applied_rule == 'user_explicit_request'
    # 否则：分格 → 1 个单元柜
    #   - 上下结构差异写入该 unit 的属性字段（has_open_shelf / open_shelf_position / drawer_count）
    #   - 旧 key_feature 'vertical_split' / 'internal_independent_doors_top_bottom' 已废弃，不构成拆分证据
    #   - 上下分段门板（panel split + 装饰腰线）按门板分段处理，仍是 1 个单元柜；若清晰横向门缝形成上下独立门板，应增加 door_count
    # 拆分约束：顶柜+底柜 必须成对生成，禁止单独出现一个

    # 步骤 B：方向判定（§2.5 边柜邻接位置偏好 / R-bay-5）
    #   仅当 remaining_bays 长度为奇数时启用方向偏好；偶数全合并双门，方向无影响。
    #   核心偏好：单门置于"远离边柜"的一端，主体大模块（双门）紧贴边柜。
    direction = 'left_to_right'  # 默认贪心从左起
    if len(remaining_bays) % 2 == 1 AND edge_cabinet_at == 'left':
        # 左侧边柜 + 奇数门柜序列 → 单门必须落在远端（最右端）
        # 正向贪心：从左到右合并双门，奇数余格自动落右端，远离左边柜
        direction = 'left_to_right'
    elif len(remaining_bays) % 2 == 1 AND edge_cabinet_at == 'right':
        # 右侧边柜 + 奇数门柜序列 → 单门必须落在远端（最左端）
        # 反向贪心：从右到左合并双门，奇数余格自动落左端，远离右边柜
        direction = 'right_to_left'
    elif len(remaining_bays) % 2 == 1 AND edge_cabinet_at == 'both':
        # 双侧夹奇数序列：单门只能在内部，缺省置于"靠右边柜"的一端
        # 同时写入 uncertainty_notes（左/右两种摆放均可）
        direction = 'left_to_right'   # 缺省单门偏右

    # 步骤 C：按 direction 横向贪心合并 remaining_bays
    units = []
    bays_iter = remaining_bays if direction == 'left_to_right' else reversed(remaining_bays)
    while not consumed_all(bays_iter):
        a, b = next_two_unconsumed(bays_iter)
        if b is not None AND adjacent_pair_has_no_split_evidence(a, b, key_features):
            units.append( Unit(bays=[a, b], bay_count=2,
                               applied_rule='preferred_double_door') )
        else:
            skip_reason = pick_explicit_reason(a, key_features)
                          or ('far_from_edge_cabinet_aesthetic'
                              if edge_cabinet_at != 'none' AND is_far_from_edge(a, edge_cabinet_at) else None)
                          or 'odd_bay_remainder'
            units.append( Unit(bays=[a], bay_count=1, applied_rule='single_door',
                               skip_reason=skip_reason) )
    return merge_then_sort_left_to_right(locked_units + units)
```

| 输入分格数 | 关键特征 | 结果 | 备注 |
|---|---|---|---|
| 2 | 等宽、无反证 | 1 个双门柜（bay_count=2） | 标准对开 |
| 2 | 一格开放柜 + 一格门柜 | 1 开放柜（bay_count=1）+ 1 单门柜（bay_count=1, `adjacent_open_shelf`） | |
| 3 | 等宽、无反证、**无边柜邻接** | 1 个双门柜 + 1 个单门柜（单门柜：`odd_bay_remainder`） | 单门位置不确定时默认置于最右，写入 `uncertainty_notes` |
| **1+3** | **左侧边柜 + 3 等宽门柜（无反证）** | **边柜 + 1 双门 + 1 单门 = 1:2:1**（单门 `far_from_edge_cabinet_aesthetic`） | **R-bay-5 触发**：单门置于远离边柜的右端 |
| **1+3** | **右侧边柜 + 3 等宽门柜（无反证）** | **1 单门 + 1 双门 + 边柜 = 1:2:1**（单门 `far_from_edge_cabinet_aesthetic`） | **R-bay-5 触发**：单门置于远离边柜的左端 |
| **1+3+1** | **双侧边柜夹 3 等宽门柜** | 边柜 + 双门 + 单门 + 边柜（缺省单门偏右）；可选 边柜 + 单门 + 双门 + 边柜 | 写入 `uncertainty_notes`，告诉用户两种均合理 |
| 1+2 | 边柜 + 2 等宽门柜 | 边柜 + 1 双门（1:2） | N=2 偶数，**R-bay-5 不触发** |
| 3 | 中间格开放柜 | 1 单门 + 1 开放 + 1 单门 | 三个 `adjacent_open_shelf` |
| 4 | 等宽、无反证 | 2 个双门柜 | |
| **1+5** | **左侧边柜 + 5 等宽门柜（无反证）** | **边柜 + 双门 + 双门 + 单门 = 1:2:2:1** | R-bay-5 触发：单门置于远离边柜的右端 |
| **1+5（含跨格抽屉）** | **左侧边柜 + 5 等宽门柜，bay 2-3 下区横跨 3 抽屉（DB1）** | **边柜 + 双门(2-3) + 双门(4-5) + 单门(6) = 1:2:2:1**（u2 `evidence=cross_bay_band:DB1`；u4 `far_from_edge_cabinet_aesthetic`） | **R-bay-7 优先锁定 bay 2-3**，剩 3 bay 走 R-bay-5 反向贪心 |
| 4 | 1+2+1 把手分布 | 1 单门 + 1 双门 + 1 单门 | 把手对称性证据 |
| 4 | 等宽 + bay 2-3 下区横跨 3 抽屉（DB1） | bay 1 单门 + 双门(2-3) + bay 4 单门 | **R-bay-7**：DB1 锁定 2-3 双门；剩 bay 1 / bay 4 各为单门 `adjacent_drawer_unit` |

> 关键纪律：决策树执行结果必须对照 `template.md §4 R-bay-1~7` 的硬规则做自检；任一违反必须重跑此步而不是放行。

#### 2.5 边柜邻接的位置偏好（强约束 / R-bay-5 关联）

> 该规则在 §2.3 决策树之上**约束合并方向**，仅当 §2.3 已决定合并双门柜且存在奇数余格时才生效。
> ⚠️ **优先级低于 §2.1 第 0 条 R-bay-7**：若 R-bay-7 已锁定部分 bay 为跨格双门柜，本规则只对**剩余未被锁定的 bay 序列**生效。

**触发条件**（须同时满足）：
1. 组合柜中存在 `type=边柜` 的单元柜，独占 1 个立面分格
2. 与边柜相邻的"剩余门柜分格序列"长度 N 为 **奇数（≥3）**
3. 序列内门柜分格之间无显式单门反证（即满足 §2.1 的优先双门条件）
4. 序列内未被 R-bay-7 锁定（已锁定的 bay 不参与本规则）

**核心约束**：序列内的 1 个 `bay_count=1` 单门柜必须**置于远离边柜的一端**，`bay_count=2` 双门柜紧贴边柜并向远端连续延展。

| 边柜位置 | N=奇数序列布局 | 等价决策树方向 |
|---|---|---|
| 左侧边柜 | 边柜 \| 双门 \| 双门 \| ... \| **单门** | `left_to_right` 贪心 |
| 右侧边柜 | **单门** \| ... \| 双门 \| 双门 \| 边柜 | `right_to_left` 贪心 |
| 双侧夹 | 边柜 \| 双门 \| ... \| **单门** \| 边柜 （缺省单门偏右） | `left_to_right`，写 uncertainty_notes |

**何时不触发（豁免）**：

- N 为偶数（包括 N=2 / 4 / 6...）：序列内无单门，方向无意义
- N=1：序列只有 1 格，本就是单门柜，无方向选择
- 边柜与序列之间还有其他单元柜插入（如另一开放柜插入中间）：此时"邻接"关系断裂，不触发
- 序列内有显式单门反证（把手对称性破缺 / 材质突变 / 抽屉柜等）：按 §2.2 规则，单门位置已被反证决定，本规则不再覆盖
- 序列已被 R-bay-7 跨格构件全部或部分锁定：被锁定的 bay 不参与方向贪心；剩余 bay 若仍构成奇数序列再启用本规则

**美学依据**：定制家具行业实践经验——边柜本身已是窄/局部小模块，与之相接的主体应保持连续大模块（双门双门）以延续视觉重量，单门作为远端收尾使立面节奏呈"小（边柜）→ 大（双门连续）→ 小（单门）"的稳定起承转合。该规则可由用户文本明确覆盖（如"我希望单门紧贴边柜"），覆盖时按 `user_explicit_request` 处理。

### 步骤 3：生成柜体单元（单元柜）基本信息

1. **单元柜类型**，严格使用以下字符串：
   - `高柜/下柜`：单段通高柜
   - `边柜`：两侧窄柜
   - `开放柜`：整柜无门、全为开放层板 / 格子
   - `顶柜`：一个立面分格如果分成两个柜体单元时，上段和底柜配对
   - `底柜`：一个立面分格如果分成两个柜体单元时，下段和顶柜配对

   > 说明：禁止写 `高柜` 单独一词 —— 必须写 `高柜/下柜`。

2. **门板数量**
3. **单元柜占的立面分格数量**：1 或 2
4. **是否有开放部分**：即柜子有部分没有门板覆盖
5. **推测宽度**
6. **推测高度**
7. **在组合中的位置**

### 步骤 4：辅助构件识别

识别顶收口板 / 踢脚装饰板 / 左右收口板或见光板的有无；记录外观朝向（WGJX）、估算总宽（H=2400、D=550 类目锁定）。

## 主要类型语义定义

### 3.0 分格内部结构 → 单元柜 type 映射表（**先查此表再选 type**）

> 一字到顶类目下，单元柜 type 由"分格观察特征 + 位置"二维决定。下表是查表优先级最高的判定依据。

| # | 分格观察特征 | 位置 | 映射 type | 关键属性 |
|---|---|---|---|---|
| 1 | 全高单 / 双门，上下统一封闭 | 任意 | `高柜/下柜` | `door_count=1或2`, `has_open_shelf=false` |
| 2 | 全高纯开放层架，无任何门 | 任意 | `开放柜` | `door_count=0`, `has_open_shelf=true`, `open_shelf_position=full` |
| 3 | 全高 + 含独立抽屉段 | 任意 | `高柜/下柜` | `drawer_count ≥ 1` |
| 4 | **上开放层架 + 下封闭门柜**（窄宽 300–600） | **最左 / 最右** | **`边柜`** | `has_open_shelf=true`, `open_shelf_position=top` |
| 5 | **上开放层架 + 下封闭门柜** | **中部（不在两端）** | **`高柜/下柜`（含开放区）** | `has_open_shelf=true`, `open_shelf_position=top` |
| 6 | 上段挂衣开放 + 下段抽屉 | 任意 | `高柜/下柜` | `has_open_shelf=true` + `drawer_count` |
| 7 | **物理独立柜体边界**（中间观察到独立柜体顶板+底板，**含柜板厚度 ≥18mm**） | 任意 | **`顶柜` + `底柜` 成对** | 必须满足前置硬约束的显式证据；上下分段门板**不**单独构成证据 |
| 8 | 用户明确指定垂直拆分 | 任意 | `顶柜` + `底柜` 成对 | `applied_rule = user_explicit_request` |

⚠️ **关键反例**（务必区分）：
- "上开放下封闭" + 在**端部** → 第 4 行（**边柜**），不是开放柜+底柜
- "上开放下封闭" + 在**中部** → 第 5 行（**高柜/下柜含开放区**），不是边柜也不是顶柜+底柜
- "上下材质突变"本身**不构成**拆分证据 → 仍按内部构造记录到属性字段
- **"上下分段门板 + 中间装饰横线"** → 属于**门板分段（panel split）**，仍是 **1 个单元柜**，**不**拆为 顶柜+底柜；但若横向门缝清晰形成上下独立门板，应按可见独立门板数增加 `door_count`
  - 极简衣柜常见做法：单柜 H=2400，门板按上 1500 + 下 900 等比例分段，中间装饰腰线/木纹横条
  - 唯一允许拆分的判据是**物理独立柜体边界**（中间观察到含柜板厚度 ≥18mm 的独立柜体顶板+底板）或用户明确指定
  - 注意：这里的“不拆成顶柜+底柜”只影响 unit 拆分，不影响门板计数；上下两块可见门板仍应分别计入 `door_count`。

### 高柜（= 下柜）
- 单段通高单元柜，从踢脚顶面延伸至顶封板底面  
- 是一柜到顶组合的主体单元，横向排列
- 高度接近组合总高（≈ H − 顶收口高 − 踢脚高）
- **可含开放区**：分格中部"上段开放层架 + 下段带门"也归为本类型（含开放区的高柜），写 `has_open_shelf=true` + `open_shelf_position=top`
- ⚠️ **重要**：在一柜到顶类目中，商品库里的"下柜"产品可直接充当高柜使用。图像中识别为高柜时，ABD 中语义类型记为 `高柜/下柜`，搜索时须同时检索"高柜"和"下柜"两类名称。

### 边柜
- **位置（必要条件）**：组合最左侧或最右侧
- **宽度**：常见 300–600 mm（窄于高柜的基础分格宽）
- **高度**：通高（与高柜相同）
- **内部构造**（不影响"1 个单元柜"判定）：边柜内部常见以下形态——
  - **上段开放层架 + 下段带门小柜**（最常见，本图属于此型）
  - 全段开放层架（虽接近开放柜，但因位于两端 + 窄宽，仍归边柜）
  - 全段带门
  - 上段带门 + 下段抽屉

**与其他类型的区分（关键反例）**：

| 特征对比 | 边柜 | 开放柜 | 高柜（含开放区） | 顶柜+底柜成对 |
|---|---|---|---|---|
| 位置 | **必须最左/最右** | 任意 | 中部为主，也可两端 | 任意 |
| 宽度 | 窄（300–600） | 不限 | 标准基础分格宽 | 不限 |
| 内部上下结构 | **可有上下不同**（上开放下封闭等） | 全开放 | 可有上开放下封闭 | **必须有独立柜体边界**（含柜板厚度 ≥18mm 的物理隔断）；上下分段门板**不**单独构成证据 |
| 单元柜数 | **1** | 1 | **1** | **2 成对** |

⚠️ **本类目下的特别提示**：
- 上开放下封闭 + 最左/最右 + 窄宽 → **边柜**（绝不识别为"开放柜+底柜垂直分段"）
- 上开放下封闭 + 中部（不在两端） → **高柜/下柜（含开放区）**

### 开放柜
- 整个单元柜**无任何门板**，门面方向是连续敞开的层板格 / 隔栅 / 抽屉外露
- 区别于"含开放格的高柜"：开放柜整体都是开放结构（门板数=0），而高柜内可能仅在某个分段是开放的，这段没有门板覆盖
- 区别于"含开放层架的边柜"：边柜的开放层架通常只占上段，下段有封闭门柜或仍为通高门；开放柜整体无门
- 典型用途：放置常用衣物、装饰品、包包、行李箱展示
- 商品命名通常含 `开放` 字样（如"开放衣柜102"），且产品规格中门板数为 0
- 行业说明：开放式柜体取放方便、视觉通透，但无防尘性能，常作为整柜组合中的"展示位"穿插在高柜之间

### 顶柜（上柜）/ 底柜（下柜）

⚠️ **在 floor-to-ceiling 类目中默认不出现**——本类目下每个立面分格默认作为 1 个单元柜处理。仅当出现以下任一**显式垂直拆分证据**时，才生成顶柜+底柜成对结构（详见前置硬约束节）：

1. 图像中分格内部存在**独立物理柜体边界**（独立柜体的顶板+底板构成横向隔断，**含柜板厚度 ≥18mm**，不是开放层板或装饰横线）
2. **用户文本明确**要求

⚠️ **强制成对**：顶柜与底柜必须**成对出现**，不允许只生成一个。

【定义保留（其他类目参考）】
- 顶柜：立面分格垂直分段时上段单元柜，典型 400–800 mm
- 底柜：立面分格垂直分段时下段单元柜，与顶柜成对出现，典型 800–1200 mm

⚠️ **不要混淆**：
- "上段开放层架 + 下段封闭"**不是**顶柜+底柜——上段无门、不是独立闭合柜体。这种形态按 §3.0 第 4 / 5 行处理
- "上下分段门板 + 中间装饰横线"**也不是**顶柜+底柜——这是门板分段（panel split）做法，仍是 1 个单元柜；门数按可见独立门板累计，参见 §3.0 关键反例
