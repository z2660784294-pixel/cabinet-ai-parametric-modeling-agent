更新时间：2026-05-13

# 2. 主要类型定义

## 单元柜：
1. 组合柜是有若干个单元柜(竖向柜体模块)组成的
2. 每个单元柜在宽度方向占1个立面分格或者2个立面分格
3. 单元柜的类型为：高柜/下柜，边柜，开放柜，其他 

## 立面分格：衣柜正立面在水平方向被主竖向分割线划分出的近似等宽竖向区间，后续进一步组合成单元柜。

## 高柜/下柜：
1. 通高单元柜，从踢脚顶面延伸至顶封板底面。是一柜到顶组合的主体单元，横向排列
2. 高柜通常是门板和抽屉，当高柜在中间的时候，可能出现开放层架(即没有门)

## 边柜：
1. 在组合柜的最左边或者最右边，宽度接近1个立面分格，有些边柜的主方向(门所在面)是组合柜的侧面
2. 当边柜门朝向组合柜侧边的时候，在边柜的侧面，可以有装饰板来修饰

## 开放柜： 整个单元柜无任何门板，门面方向全为开放层板格，高度是整个组合柜的高度

## 其他：无法归纳到高柜/下柜，边柜，开放柜 的单元柜的类型


# 3. 分析步骤

## 步骤1. 识别竖向柜体模块（unit_count）
请按"结构完整的柜体单元"识别组合柜的竖向模块数量，不要只按颜色、材质、门缝或开放格边界计数。
在一柜到顶衣柜中，竖向柜体模块即单元柜, 是一个功能柜体

1.1 先判断是否存在边柜/异形开放格模块
1. 该模块可能在左侧、右侧，也可能不存在
2. 若某一区域包含：开放格、木色内胆、弧形/圆角侧板、上下封闭柜、以及相邻白色竖向面板，并共同构成一个连续功能柜体，则**合并为 1 个边柜模块**，不要按开放格或材质边界拆分
3. 边柜的主方向(门面方向)可能朝向组合柜正面，也可能朝向侧面；当朝向侧面时，正面只有装饰板/见光板，其竖向线条**不算柜体边界**
4. 边柜内部的开放/封闭交界、木色/白色材质交界、弧形/圆角拐角**不是模块分隔线**，属于同一边柜内部的子结构

1.2 识别开放柜
1. 去除边柜后，剩下的立面分格中，检查是否有开放柜，如果有的话，开放柜单独组成一个单元柜

1.3 识别主体高柜/下柜模块
边柜,开放柜以外的主体区域，按以下证据判断模块边界：
1. 完整柜体结构（顶板/底板对齐、侧板对齐）
2. 门板组合关系（同组双门通常对称、缝隙较窄）
3. 拉手关系（同一根长拉手两侧的两扇门通常属于一个双门模块；独立长拉手通常对应一个单门或双门模块）
4. 明确的竖向分隔（柜体竖板交界、门缝明显较宽于双门内缝）
5. 连续等宽门板优先合并为双门模块（占 2 个立面分格），不要默认拆成单门模块
   - 例：3 个连续门板 → 1 双门 + 1 单门，而不是 3 个单门
   - 例：4 个连续门板 → 2 个双门，而不是 4 个单门
6. 两个连续的立面分格共有跨格抽屉/连续拉手时，强制合并为同一个双门模块

## 步骤2. 识别每个竖向柜体模块即单元柜信息
对步骤 1 得到的每个模块，
1. 按 §2 定义判型为：高柜/下柜、边柜、开放柜、其他。
2. 识别每个模块的门板的数量，对于边柜，当边柜门朝向组合柜侧边的时候，要在边柜门所在面数门的数量，即不是组合柜的正面
3. 如果有抽屉，识别抽屉的数量，如果有开放区域，识别开放搁架的数量
4. 大致估计这个单元柜的宽度和高度
5. 单元柜属性填写
5.1. type：高柜/下柜，边柜，开放柜，其他 
5.2. door_count：每个单元柜门的数量，按可见独立门板总数累计
5.3. bay_count：每个单元柜的立面分格的数量
5.4. has_open_shelf：是否含开放区域 
5.5. est_width：按合并后实际宽度估算 
5.6. est_height：通高 

## 步骤3. 按照 `abd_for_review.example.json` 定义的json的格式输出
1. 单元柜数量 N（即 unit_count_main）
2. 从左到右逐一说明每个模块：类型猜测（边柜/高柜/开放柜/其他）+ 属性

# 5. 注意点(容易错误的场景)
1. 当开放柜在组合柜的最左边或者最右边的时候，应当做为边柜
2. 两个连续的立面分格，一个立面分格是一个门，另外一个立面分格是上下两个门，这两个立面分格应该组成一个双门柜，而不是两个单门柜
3. 对于一个立面分格，对于只有门板的情况，如果中间有明显的分隔，那么应该是上下两个门板
4. 对于边柜，门一定在门面方向上，不要在边柜的侧面数门数
5. 不要把边柜内部的开放/封闭交界、木色/白色材质交界、弧形/圆角拐角当成模块边界——它们是同一边柜内部的子结构
6. 不要因为某个候选区域宽度不均，就强行把边柜拆成多个模块 
7. 主体高柜模块边界优先看"门板组合关系 + 拉手关系 + 竖向分隔"，而不是材质或颜色交界
